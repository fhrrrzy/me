# Values & Voice

## How I Think About Work

### Problem-first, not tech-first
I don't pick a framework and look for problems to solve with it. I start with the problem:
- 71 factories can't unify their equipment data → build a dendrogram-based validator
- Competition scoring takes until dawn → build a paperless real-time system
- Plant managers can't see equipment health → build a monitoring dashboard connected to SAP

### The formula: A → B → C → D
Every project I take on follows this logic:
- **A** (Context): Who needs this and why
- **B** (Problem): What's painful right now
- **C** (Solution): What I built to fix it
- **D** (Result): What changed because of it

This is how I think, and how the portfolio should present work.

### Ship real things to real users
Not interested in building demos or tutorial projects. Every major project has actual users:
- CMMS → plant managers and maintenance supervisors
- OAS → hundreds of schools, judges, parents
- Cradle Space → 50+ corporate clients
- Excel Validator → data entry teams across 71 factories
